Data & Tokenization

From raw Shakespeare text to integer sequences

The Dataset: Tiny Shakespeare

# Read the raw text file (~1.1MB)
with open('input.txt', 'r',
          encoding='utf-8') as f:
    text = f.read()

What: Loads the entire Tiny Shakespeare corpus — all of Shakespeare's works concatenated into a single ~1MB text file.

Why: Character-level models need raw text. No preprocessing, no special tokenizers — just raw characters.

Building a Character Vocabulary

# Find all unique characters in the text
chars = sorted(list(set(text)))
vocab_size = len(chars)   # 65

# Create bidirectional mappings
stoi = { ch:i for i,ch in enumerate(chars) }  # string → integer
itos = { i:ch for i,ch in enumerate(chars) }   # integer → string

# Lambda functions for encoding/decoding
encode = lambda s: [stoi[c] for c in s]
decode = lambda l: ''.join([itos[i] for i in l])

Character-level tokenizer: Unlike GPT-3/4 which use BPE (byte-pair encoding) with ~50K tokens, this model treats each character as a token. The vocabulary is just 65 characters: lowercase/uppercase letters, digits, punctuation, whitespace.

Train / Validation Split

# Encode the entire text to a tensor of integers

# First 90% → training, last 10% → validation

val_data   = data[n:]     #   111,540 chars

get_batch() — Sampling Random Chunks

def get_batch(split):
    data = train_data if split == 'train' else val_data
    # Pick random starting positions
    ix = torch.randint(len(data) - block_size, (batch_size,))
    # Stack input sequences (x) and target sequences (y)
    x = torch.stack([data[i:i+block_size] for i in ix])
    y = torch.stack([data[i+1:i+block_size+1] for i in ix])
    x, y = x.to(device), y.to(device)
    return x, y

Key insight: The target y is just x shifted right by 1. For every position, we're asking: "given all characters so far, predict the next one."

x = h e l l o   w o
y = e l l o   w o r

estimate_loss() — Evaluation

@torch.no_grad()           # disable gradient tracking
def estimate_loss():
    out = {}
    model.eval()            # switch to eval mode
    for split in ['train', 'val']:
        losses = torch.zeros(eval_iters)   # eval_iters = 200
        for k in range(eval_iters):
            X, Y = get_batch(split)
            logits, loss = model(X, Y)
            losses[k] = loss.item()
        out[split] = losses.mean()         # average over 200 batches
    model.train()           # back to train mode
    return out

Why average 200 batches? A single batch's loss is noisy. Averaging over 200 random batches gives a much more stable estimate. @torch.no_grad() saves memory by not storing intermediate values needed for backpropagation.

Part 3

Self-Attention

The core mechanism that lets tokens communicate

The Attention Intuition

Self-attention = each token creates a Query ("what am I looking for?"), a Key ("what do I contain?"), and a Value ("what info do I provide?"). Attention score = how well a query matches each key.

Head — Single Attention Head

class Head(nn.Module):
    """ one head of self-attention """

    def __init__(self, head_size):
        super().__init__()
        self.key   = nn.Linear(n_embd, head_size, bias=False)
        self.query = nn.Linear(n_embd, head_size, bias=False)
        self.value = nn.Linear(n_embd, head_size, bias=False)
        self.register_buffer('tril',
            torch.tril(torch.ones(block_size, block_size)))
        self.dropout = nn.Dropout(dropout)

Three linear projections — each maps from n_embd=384 down to head_size=64 (since 384/6 heads = 64).

register_buffer('tril', ...) — creates a lower-triangular mask matrix. This ensures the model is causal: token at position t can only see positions ≤ t, never the future.

Head — Forward Pass

def forward(self, x):
    B, T, C = x.shape           # batch, time, channels
    k = self.key(x)              # (B, T, head_size)
    q = self.query(x)            # (B, T, head_size)

    # Compute attention scores
    wei = q @ k.transpose(-2,-1)     # (B,T,hs) @ (B,hs,T) → (B,T,T)
    wei = wei * k.shape[-1]**-0.5    # scale by 1/√(head_size)
    wei = wei.masked_fill(
        self.tril[:T, :T] == 0, float('-inf'))  # causal mask
    wei = F.softmax(wei, dim=-1)     # normalize to probabilities
    wei = self.dropout(wei)

    # Weighted aggregation of values
    v = self.value(x)                # (B, T, head_size)
    out = wei @ v                    # (B,T,T) @ (B,T,hs) → (B,T,hs)
    return out

MultiHeadAttention — Parallel Heads

class MultiHeadAttention(nn.Module):
    """ multiple heads of self-attention in parallel """

    def __init__(self, num_heads, head_size):
        super().__init__()
        self.heads = nn.ModuleList(
            [Head(head_size) for _ in range(num_heads)])
        self.proj = nn.Linear(
            head_size * num_heads, n_embd)  # projection back
        self.dropout = nn.Dropout(dropout)

    def forward(self, x):
        # Run all heads in parallel, concatenate outputs
        out = torch.cat([h(x) for h in self.heads], dim=-1)
        # Project back to n_embd dimensions
        out = self.dropout(self.proj(out))
        return out

Why multiple heads? Each head can learn different "relationships" — one might track syntactic dependencies, another semantic similarity, another positional patterns. Concatenating them gives a richer representation.

Part 4

The Transformer Block

Attention + Feed-Forward with residual connections

FeedForward — The "Thinking" Layer

class FeedFoward(nn.Module):
    """ a simple linear layer followed by a non-linearity """

    def __init__(self, n_embd):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(n_embd, 4 * n_embd),   # 384 → 1536 (expand)
            nn.ReLU(),                         # non-linearity
            nn.Linear(4 * n_embd, n_embd),    # 1536 → 384 (project)
            nn.Dropout(dropout),
        )

4× expansion ratio: Following the original Transformer paper, the inner dimension is 4× the model dimension. This gives the network more capacity to learn complex functions in the "compute" phase, before projecting back down.

Attention ≈ communication. FFN ≈ computation. Attention lets tokens share information; the FFN lets each token individually process what it learned.

Block — Putting It Together

class Block(nn.Module):
    """ Transformer block: communication followed by computation """

    def __init__(self, n_embd, n_head):
        super().__init__()
        head_size = n_embd // n_head       # 384 // 6 = 64
        self.sa   = MultiHeadAttention(n_head, head_size)
        self.ffwd = FeedFoward(n_embd)
        self.ln1  = nn.LayerNorm(n_embd)   # pre-norm
        self.ln2  = nn.LayerNorm(n_embd)

    def forward(self, x):
        x = x + self.sa(self.ln1(x))      # residual + attention
        x = x + self.ffwd(self.ln2(x))    # residual + FFN
        return x

Pre-norm (GPT-2 style): LayerNorm is applied before each sub-layer, not after. This leads to more stable training.

Residual connections (x = x + ...): Allow gradients to flow directly through the network, preventing vanishing gradients in deep models. Also allows layers to learn "refinements" rather than full transformations.

Part 5

The Full GPT Model

Assembling all the pieces into a language model

GPTLanguageModel — __init__

class GPTLanguageModel(nn.Module):

    def __init__(self):
        super().__init__()
        self.token_embedding_table = nn.Embedding(
            vocab_size, n_embd)        # 65 → 384
        self.position_embedding_table = nn.Embedding(
            block_size, n_embd)        # 256 → 384
        self.blocks = nn.Sequential(
            *[Block(n_embd, n_head=n_head)
              for _ in range(n_layer)]) # 6 transformer blocks
        self.ln_f = nn.LayerNorm(n_embd)  # final layer norm
        self.lm_head = nn.Linear(
            n_embd, vocab_size)        # 384 → 65 (logits)

        self.apply(self._init_weights) # custom initialization

Full Architecture Diagram

Forward Pass & Loss

def forward(self, idx, targets=None):
    B, T = idx.shape

    tok_emb = self.token_embedding_table(idx)   # (B,T,C)
    pos_emb = self.position_embedding_table(
        torch.arange(T, device=device))          # (T,C)
    x = tok_emb + pos_emb     # combine both embeddings
    x = self.blocks(x)        # pass through 6 blocks
    x = self.ln_f(x)          # final layer norm
    logits = self.lm_head(x)  # project to vocab  (B,T,65)

    if targets is None:
        loss = None
    else:
        B, T, C = logits.shape
        logits = logits.view(B*T, C)    # flatten for CE
        targets = targets.view(B*T)
        loss = F.cross_entropy(logits, targets)

    return logits, loss

Position embeddings are learned (not sinusoidal like the original Transformer). torch.arange(T) creates [0, 1, 2, ..., T-1] — each position gets its own learned vector.

Cross-entropy loss: Compares the model's predicted probability distribution (logits) against the actual next character. Lower loss = better predictions.

Weight Initialization

def _init_weights(self, module):
    if isinstance(module, nn.Linear):
        torch.nn.init.normal_(module.weight,
                              mean=0.0, std=0.02)
        if module.bias is not None:
            torch.nn.init.zeros_(module.bias)
    elif isinstance(module, nn.Embedding):
        torch.nn.init.normal_(module.weight,
                              mean=0.0, std=0.02)

Why 0.02? This follows GPT-2's initialization scheme. Small random weights (mean=0, std=0.02) prevent large activations at the start of training. Biases start at zero. This seemingly small detail is crucial for training stability — bad initialization can make training fail entirely.

generate() — Text Generation

def generate(self, idx, max_new_tokens):
    for _ in range(max_new_tokens):
        # Crop to last block_size tokens (context window)
        idx_cond = idx[:, -block_size:]
        # Forward pass
        logits, loss = self(idx_cond)
        # Get logits for the LAST position only
        logits = logits[:, -1, :]        # (B, C)
        # Convert to probabilities
        probs = F.softmax(logits, dim=-1) # (B, C)
        # Sample from the distribution
        idx_next = torch.multinomial(
            probs, num_samples=1)         # (B, 1)
        # Append to the sequence
        idx = torch.cat(
            (idx, idx_next), dim=1)       # (B, T+1)
    return idx

Putting it all together — the training loop

The Training Loop

# Create model and move to GPU
model = GPTLanguageModel()
m = model.to(device)
print(sum(p.numel() for p in m.parameters())/1e6, 'M parameters')
# → 10.788929 M parameters

# AdamW optimizer
optimizer = torch.optim.AdamW(
    model.parameters(), lr=learning_rate)  # lr=3e-4

for iter in range(max_iters):       # max_iters = 5000
    # Evaluate every 500 steps
    if iter % eval_interval == 0 or iter == max_iters - 1:
        losses = estimate_loss()
        print(f"step {iter}: train loss {losses['train']:.4f},"
              f" val loss {losses['val']:.4f}")

    # Sample a batch
    xb, yb = get_batch('train')

    # Forward pass + compute loss
    logits, loss = model(xb, yb)

    # Backward pass
    optimizer.zero_grad(set_to_none=True)  # reset gradients
    loss.backward()                         # compute gradients
    optimizer.step()                        # update weights

Training Loss Curve

AdamW (Adam with decoupled weight decay) is the go-to optimizer for transformers. The learning rate 3e-4 is a common "good default" for small models. Training converges in ~5000 steps — about 5 minutes on a modern GPU.

Overfitting gap: The gap between train and val loss shows some overfitting — the model has memorized parts of the training data. More data or stronger regularization would help.

Architecture Summary

Data Pipeline

  • Character-level tokenizer (65 vocab)
  • 90/10 train/val split
  • Random batch sampling
  • Input → target shift by 1

Model

  • Token + position embeddings
  • 6 transformer blocks
  • 6 attention heads (64d each)
  • 384 embedding dimension

Each Block

  • Pre-LayerNorm
  • Multi-head self-attention
  • Feed-forward (4× expansion)
  • Residual connections

Training

  • Cross-entropy loss
  • AdamW optimizer (lr=3e-4)
  • 5000 iterations
  • ~10.4M parameters

Key Takeaways

1

The Transformer is simple. The entire GPT architecture fits in ~200 lines of PyTorch. The core idea — attention + feed-forward + residuals + layer norm — is elegant and powerful.

2

Attention is the key innovation. Self-attention lets every token dynamically decide which other tokens to "pay attention to" — a learned, data-dependent communication channel.

3

Scale is what made GPT work. This model and GPT-4 share the same architecture. The difference is parameters, data, compute, and fine-tuning — not fundamental design.
